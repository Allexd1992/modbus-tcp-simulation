Modbus TCP Server Simulation

MCP для ИИ

Model Context Protocol — отдельный HTTP-сервис с тем же in-memory хранилищем, что и REST: чтение/запись регистров и дискретов через tools для клиентов вроде Cursor.

  • Транспорт: Streamable HTTP, путь /mcp
  • Подсказка URL подставляется из адреса этой страницы…
  • Внутри контейнера MCP слушает 8081; с хоста при пробросе 18081:8081 порт по умолчанию в примере ниже — 18081. Иной порт: добавьте в URL UI параметр ?mcpPort=8081 (нативный запуск без Docker).
  • Переменная окружения MCP_SERVER_PORT (в процессе по умолчанию 8081); 0 — MCP выключен
  • В tools адрес регистра — смещение по протоколу (0-based): 40001 → addr 0, 40021 → addr 20.
  • Инструменты: modbus_read_holding_registers, modbus_write_holding_registers и др.

Пример конфигурации Cursor

Файл mcp.json: в каталоге проекта .cursor/mcp.json или глобально %USERPROFILE%\.cursor\mcp.json (Windows). После изменения — полный перезапуск Cursor. Хост и порт в примере берутся из адреса этой страницы (и ?mcpPort=).
